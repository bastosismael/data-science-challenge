def corrigir_base(df):
    dados = df.loc[0:7].iloc[:,0:2]
    dados.columns = dados.loc[0]
    dados = dados.loc[1:]
    dados.iloc[:,1] = [x.replace(',', '.') for x in  dados.iloc[:,1] ]
    orc = df.loc[10:]
    orc.columns = [i[1] for i in list(orc.loc[10].items())]
    orc = orc.loc[12:]
    c = [j for j in orc.columns if j == j]
    cols_id = [j for j in c if not str.startswith(str(j), "Pre")]
    precos = [j for sub in [(a + "_Unitario", a + "_Total") for a in c if str.startswith(a, "Pre")] for j in sub]
    orc.columns = cols_id + precos
    # Transformando colunas de preos e Quantidade
    orc['Quantidade'] = [x.replace(',', '.') if x == x  else x for x in  orc['Quantidade'] ]
    orc['Quantidade'] = orc['Quantidade'].astype(float)
    for c in precos:
        orc[c] = [x.replace('R$ ', '') if x == x  else x for x in  orc[c]]
        orc[c] = [x.replace('.', '') if x == x  else x for x in  orc[c] ]
        orc[c] = [x.replace(',', '.') if x == x  else x for x in  orc[c] ]
        orc[c] = orc[c].astype(float)
    return (dados, orc)

A funo definida acima recebe a base em seu estado bruto e gera duas novas bases, uma com informaes gerais do projeto e outra contendo informaes acerca do oramento. Alm disso, realiza os seguintes ajustes:

  • Remove o R$ das colunas de preo
  • Substitui o separador decimal de "," para "."
  • Remove o separador de unidades de milhar
  • Converte todas as colunas numricas para o tipo Float.

Como pode ser visto acima, no parece haver evidncias de que projetos da tipologia sobrado apresentam elementos epecficos quando comparados ao projeto da tipologia Trrea. importante ressaltar que essa anlise desconsidera todas as demais variveis que poderiam afetar a presena e ausncia de elementos. Apesar disso, devido ao Projeto 4 apresentar tipologia Sobrado, adotou-se as amostras 2 e 3 como referncia e o processo de estimao do oramento para o Projeto 4 se dar da seguinte forma:

  • $M(X_1, X_2, X_3)$ caso o item analisado esteja presente nas trs amostras.
  • $M(X_2, X_3)$ caso o item analisado esteja presente apenas nas amostras $3$ e $2$.
  • $M(X_2)$ caso o item analisado esteja presente apenas na amostra $2$.
  • $M(X_3)$ caso o item analisado esteja presente apenas na amostra $3$.

Em que, $M$ o modelo que ser definido na seo seguinte e $X_i, i\in \{1,2,3\}$ representa o conjunto de covariveis referentes ao elemento analisado do projeto $i$. De forma a simplificada a intuio por trs da regra acima apresentado no exemplo abaixo:

Tomemos como exemplo o item CONSUMO DE AGUA DURANTE A OBRA, esse elemento est presente nos trs projetos de amostra, logo a sada da estimativa para o Projeto 4 ser dada por:$$M(X_1, X_2, X_3)$$

Onde cada $X_i$ representa o vetor definido da seguinte forma: $X_i = (\text{Tipologia}_i, \text{Area Terreno}_i, \text{Area Construda}_i, \text{Area Fundacao}_i, \text{Area Fachada}_i, \text{Area Parede}_i, \text{Qtde BWCs}_i, \text{Quantidade}_i), \forall i \in \{1,2,3\}$

Agora tomemos o item CONCRETAGEM E DESFORMA - VIGA DE BORDO; 20X40CM, esse elemento est presente nos projetos $2$ e $3$, mas no est presente no projeto $1$, nesse caso a estimativa desse item para o Projeto 4 ser dada por: $$M(X_2, X_3)$$

De fato, a abordagem proposta acima indica Superestimao do custo, uma vez o conjunto de elementos presentes no oramento do Projeto 4 ser dado por: $$\left(\cap_{i=1}^{3}E_i\right) \cup (E_2\cap E_3 \cap E_1^c) \cup (E_2 \cap E_3^c \cap E_1^c) \cup (E_3 \cap E_2^c \cap E_1^c)$$

Onde $E_i$ representa o nmero de elementos no oramento do projeto $i$, com $i \in \{1,2,3\}$

Modelagem

Uma das limitaes evidentes do presente desafio o fato de estarmos lidando com um problema com amostra de tamanho pequeno, o que acaba limitando a possibilidade de uso de modelos estatsticos. Apesar de limitar, isso no significa que seja impossvel, ainda existe a possibilidade de uso de tcnicas de reamostragem ou modelos que consigam enfrentar tal problema. Uma possibilidade latente o uso de algum aproximador universal de funo para estabelecer ligao entre os dados e o oramento, entretanto, como pode ser visto em Shwartz-Ziv e Armon (2022), para dados tabulares, que exatamente a estrutura na qual estamos trabalhando, o uso de tal aproximao no se mostra efeitva. Ainda em relao ao artigo supracitado, os autores reforam a ideia de que modelos baseados em rvores so mais indicados para esse tipo de dado, nesse sentido, optou-se por utilizar rvores de regresso.

O modelo proposto ser definido da seguinte forma: $$ M(X) = \begin{cases} X & \text{se} &|X| = 1\\ AD(X) & \text{se} &|X| > 1\\ \end{cases}$$

Em que $|X|$ representa o nmero de elementos no conjunto $X$

Ou seja, o modelo assume a funo identidade caso a base contenha apenas uma linha, isto , o modelo replica a informao do projeto anterior. Caso contrrio usada uma arvore de deciso para estimar o valor do projeto 4. Ademais, segue que a quantidade do item no projeto 4 ser definida como: $$ q_4 = \begin{cases} \dfrac{1}{n}\sum_{i=1}^{n} q_i & \text{se}& n \in \{1,2,3\}\\ \end{cases}$$

Ou seja, para as bases que tiveram mais de duas linhas, a quantidade do item no projeto 4 ser estimada pela mdia ou simplesmente replicado, caso tenha apenas uma linha.

Para o uso de rvores de regresso optou-se por usar a biblioteca sklearn, uma vez que trata-se de uma biblioteca bastante utilizada e largamente testada.

Concluso

O presente projeto teve por objetivo produzir uma estimativa do custo para o Projeto 4, dessa forma, possvel afirmar que o objetivo foi atingido. No entanto, algus pontos ainda necessitam ser explorados. Um dos maiores desafios do presente projeto o fato de terem sido apresentadas apenas trs amostras, o que acaba tornando difcil o uso de modelos mais robustos ou que se baseiam em estimativa de funes de aproximao, e at mesmo se tornando impossvel verificar validade de pressupostos de alguns modelos estattsticos clssicos. Nesse sentido, a abordagem apresentada no texto visou explorar o algortmo de rvore de regresso, uma vez que, de maneira simplificada, o algoritmo funciona como uma espcie de tomador de deciso que consegue tomar decises olhando para todas as amostras ao mesmo tempo.

Em relao a previso da incerteza, no foi possvel identificar formas de estimar a incerteza das previses, uma vez que a construo de intervalos de confiana exigiria a verificao de caractersticas dos dados que no so possveis de verificar com uma amostra to pequena. Outra possibilidade seria a utilizao de previso conformal, mas acabaramos incorrendo no mesmo problema. De maneira geral, pode-se afirmar que a estimativa do custo razovel, uma vez que pela natureza do modelo proposto, todas as estimativas esto dentro do range histrico para cada um dos elementos do oramento.

Uma possibilidade que tambm poderia ter sido usada a adoo de Redes Bayesianas, que um modelo que visa estudar a estrutura de dependncia entre as variveis analisadas, sendo classificado como um modelo colaborativo e que por sua caracterstica colaborativa permite trabalhar com amostras pequenas, no entanto, seria impossvel seu uso no presente trabalho, uma vez que no h a presena de um especialista durante o desenvolvimento do projeto.
